Параметры запроса, которые добавляет express фрейморк.

req.params - массив параметров маршрутизации
req.query - объект содержащий параметры строки запроса (GET параметры)
req.body - объект содержащий параметры POST
req.route - информация о текущем совпадающем маршруте
req.cookies / req.signedCookies - объекты, содержащие значения cookies, передаваемые от клиента
req.headers - заголовки запроса полученные от клиента
req.accepts(types) - удобный метод для принятия решения о том, должен ли клиент принимать данные тип или типы.
req.ip - IP адрес клиента
req.path - путь запроса (без протокола, хоста, порта или строки запроса)
req.hostname - возвращает переданное имя клиента
req.xhr - true если запрос выполнен с помощью AJAX
req.protocol - протокол (http / https)
req.secure - true если соединение безопасное
req.url/req.originalUrl - возвращает путь и строку запроса

Параметры ответа, которые добавляет express фрейморк.

res.status(code) - устанавливает код состояния HTTP. По умолчанию 200.
res.set(name, value) - устанавливает заголовок ответа.
res.cookie[name, value, [options]], res.clearCookie(name, [options]) - устанавливает или очищает куки.
res.redirect([status], url) - выполняет перенаправление браузера. По умолчанию код 302.
res.send(body) - отправляет ответ клиенту.
res.json(json) - отправляет JSON клиенту.
res.jsonp(json) - отправляет JSONP клиенту.
res.end() - завершает соединение без отправки ответа.
res.type(type) - устанавливает заголовок Content-Type. 
res.format(object) - позволяет отправлять различное содержимое в зависимости от заголовка Accept запроса.
res.attachment([filename], res.download(path, [filename], [callback])) - оба метода устанавливают заголовок ответа Content-Disposition в значение attachment. Это указывает браузеру что необходимо загружать содержимое вместо его отображения.
res.sendFine(path, [options], [callback]) - этот метод читает файл, заданный параметром path, и отправляет еего содержимое клиенту.
res.links(link) - задает заголовок ответа Links. Редко используеемый.
res.locals, res.render(view, [locals], callback) - res.locals - объект содержащий контекст "по умолчанию" для рендеринга представлений.
res.render - отображает представление используя указанный в настройках шаблонизатор.
