В веб разработке качество может рассматриваться в четырех аспектах:

- Охват (SEO)
- Функциональность
- Удобство использования (Human-computer interaction, HCI)
- Эстетика

/**********/

Логика и визуализация:

- Логика - модули JS
- Визуализация - HTML + CSS + JS + React/Vue/Angular

/**********/

Виды тестов:

- Модульные - тестирование отдельных компонентов
- Интеграционные - взаимодействие многих компонентов или системы целиком

/**********/

Методы:

- Модульное тестирование - тестирование найменьших отдельных компонентов. Обычно отдельных функций. Jest.
- Интеграционное тестирование - охватывают большие функциональные еденицы. Puppeteer + Jest.
- Линтинг - нахождение участков потенциальных ошибок / нестабильных компонентов. ESLint.

/**********/

Установка Jest:

npm install --save-dev jest (--save-dev - означает что эта зависимость нужна только для разработки, приложение может функционировать без нее)

/**********/

Mock-объекты:

Что бы тестирование было эффективным, зависимости нужно имитировать mock-объектами.

/**********/

проверка на сколько протестирован код npm test -- --coverage

/**********/

Установка Puppeteer + Portfinder:

npm install --save-dev puppeteer
npm install --save-dev portfinder

/**********/

Установка ESLint:

npm install --save-dev eslint

Запуск утилиты для формирования файла конфигурации:

./node_modules/.bin/eslint --init
